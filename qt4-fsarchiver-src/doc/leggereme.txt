Il trattamento dei qt4 - FSArchiver è semplice:

Utilizzando il menu opzioni "Azioni" dalla barra degli strumenti oppure possono essere selezionate le seguenti azioni :
"Backup partizione / ripristinare le partizioni ' , ' backup della partizione / ripristino su una rete " , "backup MBR / GPT / ripristinare i file " , " Directories Salva / Ripristina " e " clone HD , creare o ripristinare all'Immagine " .
Il programma rileva automaticamente se il Master Boot Record o tabella di partizione GUID devono essere protetti .
Inoltre, le voci di menu "Impostazioni" impostazioni di base: come il numero di core o di resistenza alla compressione e possono essere regolati ulteriormente .
Quando i backup delle partizioni possono essere utilizzati per deterministicamente miniera Sia il record di avvio della partizione ( PBR ) shoulderstand essere sostenuto Grub non supporta il PBR . Con Grub , il backup è in genere alcun significato .

Il backup è sempre seguito lo stesso schema : Si sceglie il nome per il backup , il programma aggiunge automaticamente il nome con l'indicazione della data , poi si sceglie la partizione di cui eseguire il backup o la directory per il backup Ora la directory vengono selezionati in Quale shoulderstand memorizzato il backup. quindi selezionare la directory in cui Shoulderstand archiviato il backup. il Tasto " partizione sicura " e iniziare il backup .

Il ripristino è simile : Nella directory di backup viene selezionato il nome del file da ripristinare . Quando si ripristina una partizione da ripristinare la partizione è selezionata . Si valuta se il nome della partizione di ripristino ( es. / dev/sda5 ) , corrisponde al nome della partizione protetta. Dopo aver ricevuto l'avviso , la partizione può essere ripristinato in qualsiasi posizione . Se esiste un file con il settore di avvio della partizione come contenuto , viene chiesto se la PBR è da ristrutturare .
Quando si ripristina una btrfs Partiiton è possibile scegliere se l' shoulderstand partizione formattata o meno. Quando si formatta viene generato un nuovo UUID Se il formato non è l' UUID Born battezzato

Durante il ripristino di directory è selezionata, se la directory Shoulderstand essere ripristinato nella posizione originale o in un'altra posizione . Premere il pulsante "Ripristina partizione" e la partizione o directory in fase di restauro .
Requisiti per il lavoro con la rete :
Quando il backup su una rete , a ulteriore forma viene aperto , in cui il nome utente e la password devono essere inseriti per la rete . Di memorizzare la password non è raccomandato al di fuori del settore privato , poiché la crittografia della password viene eseguita in modo semplice .

Con il samba - protocollo :
Se viene utilizzata la rete deve essere installato su entrambi i computer ( client e server) Samba . Client è il computer ha fatto esegue il backup / ripristino . Server è il computer dove sono memorizzati i dati .
L'integrazione dei file system " vfs CIFS " virtuali deve essere possibile sulla macchina Linux . Si prega di fare riferimento al wiki di Ubuntu articolo " http://wiki.ubuntuusers.de/Samba_Client_cifs " .
Per l'accesso a una cartella condivisa di Windows 7 è necessario aggiungere nome utente o gruppo " Everyone" in Proprietà > Condivisione> Condivisione avanzata > Autorizzazioni. Come le autorizzazioni per " Everyone " dovrebbe di " pieno accesso " , " Change" e "lettura" può essere attivato.

Con il protocollo SSH - (custodito comune):
Nella zona di Linux deve essere openssh - server installato sul computer sul quale sono memorizzati i dati ( server) Per accedere al giugno necessità singolo server di essere entrato una volta nel terminale per identificare i comandi di sintesi . :
sudo ssh user @ indirizzo del server . (Esempio : sudo ssh peter@192.168.2.10 ) . La chiamata deve essere effettuata con diritti amministrativi. Confermare la questione della impronta digitale della chiave RSA di sì. Immettere la password per accedere al server . Se questo è successo avete con i contatti di collegamento al server . Questo è indicato dalle cambiato prontamente . Lasciare il server con il comando exit . Questa procedura deve essere eseguita una sola volta per tutti i server .
La prima volta che l' accesso al programma qt4 - FSArchiver su un server , il nome utente ssh indirizzo @ server comando sudo è inserito direttamente nel terminale . Nelle impostazioni di base del programma possono essere impostati se l' ingresso automatico per accedere a server aggiuntivi shoulderstand essere ripetuta o meno.
Per il montaggio di un server ssh , il terminale deve essere la password per accedere al server per essere inserito . Per evitare questo , si consiglia di impostare una chiave pubblica . Importante: la chiave pubblica del server (file : authorized_keys ) ha bisogno in / root / ssh da copiare. .
Nelle impostazioni di base è possibile stabilire se il riferimento per impostare la password per essere visualizzato o meno.

Solo una macchina Linux come client o server può essere utilizzato . Deve essere installato sul nfs - kernel - server server. Il backup di partizioni montate è possibile dopo aver chiesto conferma . Ripristino delle partizioni possono essere effettuate solo se la partizione è montata .
Il restauro di una radice o di una partizione casa è possibile con il CD qt4 - FSArchiver dal vivo ( http://sourceforge.net/projects/qt4-fsarchiver/files/Live-CD/ " ) .
Sul SystemRescueCd è il programma disponibile FSArchiver . Con FSArchiver è radice posible e home- partizioni è possibile ripristinare .

Quando il backup del MBR deve solo selezionare ( con più dischi rigidi ) , il disco rigido e la directory di backup . L' area nascosta del disco tra la fine del MBR e l'inizio della prima partizione verrà eseguito il backup Mentre il ripristino del MBR con più dischi rigidi , il disco rigido può essere selezionato , il MBR verrà ripristinato . Inoltre , è possibile scegliere se la zona bootloader , la tabella delle partizioni o la zona nascosta del disco da ripristinare .

qt4 - FSArchiver può eseguire il backup e ripristino con ecryptfs criptati home directory . Questo vale indipendentemente dal fatto directory home dell'utente viene memorizzato in una partizione separata o all'interno della directory principale.

Con partizioni cifrate TrueCrypt non è possibile eseguire il backup

Protezione di una partizione FAT di Windows non è supportato . Tuttavia, può essere memorizzato su una partizione FAT . Il file di backup verrà diviso se la dimensione del file è maggiore di 4 GB .

Clonare hard disk : Scegliete nella casella di riepilogo di sinistra , selezionare l' unità esistente è fatto da clonare , e la casella di riepilogo di destra , l'hard disk , che deve essere ricostruito. Fare clic sul pulsante " Clona disco " .
Immagine del disco rigido creare :
Selezionare nella casella di riepilogo a sinistra del disco rigido esistente , dalla quale dell'immagine è da creare . Scegliere una directory di backup . Inoltre , selezionare se l' immagine viene compressa . Fare clic sul pulsante " dell'immagine creare " .
Immagine del disco di ripristino :
Selezionare nella casella di riepilogo a sinistra dal disco rigido , dal quale di immagine deve essere scritto indietro . Selezionare il file di backup esistente (che termina img.fsa o gz.fsa ) . Fare clic sul pulsante " immagine di ripristino " .
All'avvio qt4 - FSArchiver non sono tutte le partizioni montate sotto / media / sda ... / media / sdb ... montato . Le partizioni sono facilmente raggiungibili sul desktop . La directory in cui il backup Shoulderstand essere salvato o la directory in cui è memorizzata la partizione protetta, può essere seleczed in qt4 - FSArchiver in / media / sd ....
